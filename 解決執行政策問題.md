# 解決 PowerShell 執行政策問題

## 問題說明

PowerShell 預設禁止執行腳本，這是 Windows 的安全設定。

## 解決方案

### 方案 1: 修改執行政策（推薦，一次設定永久有效）

在 PowerShell 中執行（以管理員身份執行 PowerShell）：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

輸入 `Y` 確認。

**說明**：
- `RemoteSigned`: 允許執行本地腳本和已簽名的遠程腳本
- `CurrentUser`: 只影響當前用戶，不需要管理員權限

### 方案 2: 使用 CMD 命令提示字元（不需要修改設定）

不使用 PowerShell，改用 CMD：

1. 開啟 **命令提示字元**（CMD）
2. 執行：
```cmd
cd C:\Users\FAE\issue-system\backend
.venv\Scripts\activate.bat
```

### 方案 3: 直接使用 Python（不激活虛擬環境）

如果虛擬環境已正確安裝，可以直接使用完整路徑：

```powershell
# 使用虛擬環境中的 Python
.venv\Scripts\python.exe manage.py makemigrations
.venv\Scripts\python.exe manage.py migrate
.venv\Scripts\python.exe manage.py runserver 0.0.0.0:8000
```

### 方案 4: 臨時繞過（僅當前會話）

```powershell
powershell -ExecutionPolicy Bypass -File .venv\Scripts\Activate.ps1
```

## 推薦流程

1. **執行方案 1**（修改執行政策，一次設定）
2. 然後正常使用：
```powershell
.venv\Scripts\Activate.ps1
python manage.py makemigrations
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```
